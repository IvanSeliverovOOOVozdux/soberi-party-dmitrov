# Soberi Party — Шаблон №3 (творческая студия / мастер-классы)

Лендинг творческой студии Soberi Party (Дмитров). Сделан как **Шаблон №3** в портфолио агентства — одновременно реальный клиентский сайт и заготовка под форк под аналогичные ниши.

**Прод:** https://soberi-party-dmitrov.vercel.app/

## Чем этот шаблон отличается от №1 и №2

- **Шаблон №1 (адвокатский)** — деловой, многостраничный (3 страницы), сдержанная палитра.
- **Шаблон №2 (Санин)** — см. его репозиторий.
- **Шаблон №3 (этот)** — **одностраничник**, soft-editorial / craft-luxury стиль, тёплая палитра, декоративный слой повсюду, акцент на эмоции и крафт. Структура **принципиально другая**, чтобы портфолио не выглядело шаблонно.

Подходит под форк для: мастер-классы, творческие студии, бутики, кондитерские, флористика, керамика, свадебные сервисы, beauty-сервисы с «душевной» подачей.

## Стек

- Чистый HTML + CSS + ванильный JS. Один файл `index.html`, без сборки.
- Шрифты: Google Fonts (Fraunces, Geist).
- Деплой: Vercel, автодеплой из ветки `main`.

## Структура страницы

```
1. Header (sticky)              — лого + якорная навигация + CTA
2. Hero                          — лого-маска с SVG-анимацией, лид, две CTA, факты
3. Strip                         — бесконечная марка-строка с тегами
4. Numbers                       — 4 ключевых числа
5. Formats                       — горизонтальный rail с карточками услуг
6. Flow                          — 4 шага «как проходит»
7. Reviews                       — асимметричная сетка из 4 отзывов
8. Booking                       — заголовок + контакты + форма заявки
9. Footer                        — контакты + копирайт
```

Декоративный слой **«трилистники»** (`.petal-field` в каждой секции) — фирменная фишка шаблона, генерируется JS-ом.

## Как форкать под нового клиента

### 1. Палитра — `:root` в `<style>`

```css
--milk:    #fbf6f2;   /* основной фон, светлый */
--blush:   #f7e6e0;   /* мягкая подложка, чередующиеся секции */
--petal:   #fad7e0;   /* акцентный фон (booking) */
--pink:    #f4a8c7;   /* акцент 1, декор, чипы */
--rose:    #e88aa5;   /* акцент 2, italic-заголовки */
--mauve:   #b06479;   /* hover для тёмной кнопки */
--cocoa:   #3a2e2a;   /* основной текст, тёмная кнопка, футер */
--taupe:   #8a7a72;   /* приглушённый текст, eyebrow */
--line:    #efe1d8;   /* границы, разделители */
```

Замени все девять → перекраска целиком. Правило: `milk` светлее `blush` светлее `petal`, `pink` — основной акцент, `rose` — насыщеннее для текста, `mauve` — тёмный вариант акцента, `cocoa` — почти-чёрный для контраста.

### 2. Шрифты — две переменные

```css
--serif: 'Fraunces', Georgia, serif;       /* заголовки, italic-акценты */
--sans:  'Geist', ui-sans-serif, system-ui;/* основной текст */
```

Не забудь поменять `<link href="https://fonts.googleapis.com/css2?...">` в `<head>` под новые семейства.

### 3. Лого

- `brand/logo-wide.png` — основной логотип, используется в шапке, hero (как маска) и футере.
- В hero лого собирается из прямоугольных масок (см. `<mask id="hero-mask">` в `index.html`) — это сделано вручную под форму конкретного логотипа. **Если форкаешь — либо подгони координаты `<rect>` под новый логотип, либо замени всю секцию `.hero-logo-wrap` на простой `<img>`.**

### 4. Контент

- Тексты, цены, контакты — прямой поиск-замена по `index.html`.
- Заглушки фото: `https://picsum.photos/seed/...` → меняй на реальные фото клиента (в `brand/` или в `public/`).
- Форма (`<form class="form" ...>`): сейчас `action="mailto:..."`. Для прода замени на endpoint (Formspree, Web3Forms, Vercel serverless `/api/booking`).

### 5. Декор «трилистники»

В `<script>` есть IIFE, который генерит лепестки по всем секциям с классом `.has-petals`.

- **Убрать совсем:** удалить класс `has-petals` со всех секций — JS просто не найдёт `.petal-field`.
- **Поменять форму:** редактировать строку `const P = 'M50 50 C 37 39 ...'` и `<svg viewBox=...>` внутри IIFE.
- **Изменить плотность:** атрибут `data-petals="N"` на каждом `.petal-field`.
- **Цвет:** наследуется от `var(--pink)` — поменяется автоматически с палитрой.

## Деплой

```bash
git push origin main          # автодеплой на Vercel
```

Vercel настроен как Other / static / no build. Output — корень репо.

Под нового клиента: форкнуть репо → импортировать в Vercel под новым именем → подключить домен.

## Что в репо НЕ нужно для прода

- `IMG_0739..0742` — исходники логотипа от клиента.
- `mask-logo.js`, `split-logo.js`, `trace-logo.js`, `package.json`, `package-lock.json` — разовые node-скрипты, которыми готовили SVG-маски лого. Можно удалить при форке.
- `brand/letters/`, `brand/letters.json`, `brand/letters-masked.svg` — побочные артефакты той же подготовки логотипа.

Для прода нужны только: `index.html` + `brand/logo-wide.png` + `brand/logo-square.png`.
